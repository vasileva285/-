# ER-Диаграмма базы данных EnglishApp

## 📊 Файл диаграммы

Файл `database_er_diagram.drawio` содержит полную ER-диаграмму базы данных приложения.

## 🎨 Описание диаграммы

### Таблицы базы данных

1. **auth_user** (желтый)
   - Django встроенная таблица пользователей
   - Содержит информацию о пользователях системы

2. **lessons_lessoncategory** (зеленый)
   - Категории уроков (Present, Past, Future)
   - Уникальный slug для URL

3. **lessons_lesson** (синий)
   - Уроки с теоретическим материалом
   - Связана с категорией через FK

4. **lessons_exercise** (фиолетовый)
   - Упражнения для уроков
   - Связана с уроком через FK
   - Содержит JSON поле для вариантов ответов

5. **lessons_userprogress** (красный)
   - Прогресс пользователей по урокам
   - Уникальная комбинация (user_id, lesson_id)
   - Связана с User и Lesson

6. **lessons_userexerciseattempt** (оранжевый)
   - Попытки выполнения упражнений
   - Связана с User и Exercise
   - Хранит все попытки пользователей

## 🔗 Связи (Relationships)

### One-to-Many (1 → N)

1. **User (1) → UserProgress (N)**
   - Один пользователь имеет прогресс по многим урокам
   - Связь через `user_id` FK

2. **User (1) → UserExerciseAttempt (N)**
   - Один пользователь делает много попыток
   - Связь через `user_id` FK

3. **LessonCategory (1) → Lesson (N)**
   - Одна категория содержит много уроков
   - Связь через `category_id` FK

4. **Lesson (1) → Exercise (N)**
   - Один урок содержит много упражнений
   - Связь через `lesson_id` FK

5. **Lesson (1) → UserProgress (N)**
   - Один урок имеет прогресс от многих пользователей
   - Связь через `lesson_id` FK

6. **Exercise (1) → UserExerciseAttempt (N)**
   - Одно упражнение имеет много попыток
   - Связь через `exercise_id` FK

## 🎨 Цветовая схема

- **Желтый** (#fff2cc) - auth_user (Django встроенная)
- **Зеленый** (#d5e8d4) - lessons_lessoncategory
- **Синий** (#dae8fc) - lessons_lesson
- **Фиолетовый** (#e1d5e7) - lessons_exercise
- **Красный** (#f8cecc) - lessons_userprogress
- **Оранжевый** (#ffe6cc) - lessons_userexerciseattempt

## 📋 Обозначения полей

- **PK** (красный фон) - Primary Key (Первичный ключ)
- **FK** (синий фон) - Foreign Key (Внешний ключ)
- **UQ** (зеленый фон) - Unique (Уникальное значение)

## 🔧 Ограничения

### Уникальные индексы:
- `lessons_lessoncategory.slug` - уникальный URL категории
- `lessons_lesson.slug` - уникальный URL урока
- `lessons_userprogress(user_id, lesson_id)` - один прогресс на урок для пользователя

### Индексы для производительности:
- Все внешние ключи (FK) проиндексированы
- `attempted_at` в UserExerciseAttempt для быстрой сортировки

### CASCADE DELETE:
- При удалении категории удаляются все уроки
- При удалении урока удаляются все упражнения
- При удалении пользователя удаляется весь его прогресс и попытки

## 📐 Как открыть диаграмму

1. Откройте [draw.io](https://app.diagrams.net/) в браузере
2. Файл → Открыть → Выберите `database_er_diagram.drawio`
3. Или перетащите файл в окно draw.io
